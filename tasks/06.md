Objetivo: Implementar a Infraestrutura Global da API: Tratamento de Erros Centralizado e Middleware de Autenticação Funcional.

Contexto: A arquitetura base (Vertical Slicing) foi estabelecida. Agora, precisamos que o Fastify capture e formate corretamente os erros de validação do Zod e que crie um mecanismo reutilizável para extrair o userId de requisições autenticadas.

Tarefas de Infraestrutura:

Global Error Handler: Em src/shared/http/server.ts (ou no arquivo principal do Fastify), configure o app.setErrorHandler. Ele deve interceptar erros de validação do Zod (hasZodFastifySchemaValidationErrors), formatá-los em um objeto amigável (ex: { error: 'Validation Error', issues: [...] }) e retornar status 400. Para outros erros, registre no console e retorne 500.

Tipagem do Request (Fastify): Crie um arquivo src/shared/types/fastify.d.ts para fazer um module augmentation no Fastify. Adicione a propriedade userId: string ao tipo FastifyRequest.

Auth Hook (Middleware): Crie src/shared/http/hooks/verify-auth.ts. Escreva uma função pura e assíncrona que receba request e reply. Ela deve (neste momento inicial) simular a extração de um token dos headers/cookies e injetar um userId fictício ou de teste no request.userId. Se o token não existir, retorne reply.status(401).send({ error: 'Unauthorized' }).
