Épico: Implementar a Fase 2 (Domínio de Taxonomia: Categories, Cost Centers e Tags) para a API do Nummus Premium.
Contexto: Estamos construindo uma API Fastify usando Programação Funcional (Zero Classes), Zod para validação, e Screaming Architecture (Fatiamento Vertical por módulos).

Plano de Execução (Execute sequencialmente):

1. Módulo de Categorias (src/modules/categories/):

DTO: Crie dtos/create-category.dto.ts com um schema Zod exigindo name, color, icon e um parentId opcional.

Repository: Crie repositories/category.repository.ts. Exporte funções puras para buscar categoria por nome/parentId e para criar a categoria.

Use Case: Crie use-cases/create-category.use-case.ts. Regra de Negócio Crítica: O usuário não pode ter duas categorias com o mesmo nome no mesmo nível hierárquico (mesmo parentId). Valide isso antes de salvar.

Routes: Crie http/category.routes.ts. Registre a rota POST /categories protegida pelo nosso hook verifyAuth. Use um presenter se necessário para formatar a saída.

2. Módulo de Centros de Custo (src/modules/cost-centers/):

Crie a estrutura funcional padrão (DTO, Repository, Use Case, Routes) para criar um Cost Center.

Regra de Negócio: Impeça nomes duplicados para o mesmo usuário.

Registre a rota POST /cost-centers protegida com verifyAuth.

3. Módulo de Tags (src/modules/tags/):

Crie a estrutura funcional padrão (DTO, Repository, Use Case, Routes) para criar uma Tag.

Regra de Negócio: Impeça nomes duplicados para o mesmo usuário.

Registre a rota POST /tags protegida com verifyAuth.

4. Integração das Rotas (src/shared/http/server.ts):

Importe e registre os três novos arquivos de rotas no servidor principal do Fastify, garantindo que o Swagger os reconheça (usando tags como ['Categories'], ['Cost Centers'], ['Tags']).

Regras Absolutas para este Fluxo:

Zero Classes: Apenas paradigma funcional em todas as camadas. Injeção de dependência via High-Order Functions.

Clean Code e Idioma: Todo o código fonte gerado (variáveis, funções, nomes de arquivo) DEVE ser 100% em Inglês, sem comentários supérfluos.

Validação Contínua: Após alterar os arquivos, rode npx tsc --noEmit internamente para garantir que as tipagens (como a do request.userId) não quebraram. Conserte autonomamente se quebrar.
