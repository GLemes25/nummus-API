**Objetivo:** Configurar o provedor de autenticação do Google e expor as rotas de usuários/login no Swagger usando o Better Auth.
**Contexto:** Preciso que a documentação da API (Scalar/Swagger) exiba todas as rotas de autenticação (login, registro, sessão) para o frontend consumir, e quero habilitar o login social com o Google.
**Tarefas:**

1. Atualize o arquivo de configuração de variáveis de ambiente (`src/lib/env.ts` e o `.env.example`) para incluir `GOOGLE_CLIENT_ID` e `GOOGLE_CLIENT_SECRET`.
2. Atualize o arquivo `src/lib/auth.ts`:

- Adicione a configuração do Google em `socialProviders`.
- Importe e ative o plugin `openAPI()` do pacote `better-auth/plugins`. Exemplo: `plugins: [openAPI()]`.

3. Verifique o arquivo `src/server.ts` (ou `index.ts`) e certifique-se de que a configuração do `fastifyApiReference` (Scalar) possui uma "source" apontando para a documentação do Better Auth. A URL deve ser `/api/auth/open-api/generate-schema`.
4. Crie uma rota simples em `src/routes/users.ts` chamada `GET /me` (lembre-se de registrar no server). Esta rota deve usar a sessão do Better Auth para retornar os dados do usuário autenticado atual, servindo como validação para o frontend. Não esqueça de adicionar a propriedade `schema` (com `tags: ['Users']`) para que ela apareça na documentação principal.

**Regras do Projeto:**

- Mantenha o paradigma estritamente funcional (sem classes).
- Ao commitar, a mensagem do commit deve ser obrigatoriamente escrita em **Inglês**.
