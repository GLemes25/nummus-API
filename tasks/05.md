Contexto: O schema do Prisma já está pronto. Precisamos criar a base da API focada em testabilidade. O fluxo escolhido para inaugurar a arquitetura é a criação de uma Carteira (Wallet), agrupando todos os seus arquivos dentro do seu próprio domínio de negócio (src/modules/wallets/).

Tarefas de Instalação e Setup:

Instale as dependências para documentação e validação:
npm i @fastify/swagger @fastify/swagger-ui zod fastify-type-provider-zod

Configure o Swagger no arquivo principal do Fastify (mova ou crie em src/shared/http/server.ts ou mantenha em src/server.ts), registrando o @fastify/swagger e @fastify/swagger-ui (com a rota /docs), e configure o validator compiler e serializer do fastify-type-provider-zod.

Tarefas de Arquitetura (Fluxo da Wallet - Refatoração de Pastas):

DTO: Crie ou mova a validação para src/modules/wallets/dtos/create-wallet.dto.ts. Exporte um Zod schema chamado createWalletSchema contendo name (string), currency (opcional, default 'BRL') e initialBalance (number, default 0).

Repository: Crie ou mova a lógica de banco para src/modules/wallets/repositories/wallet.repository.ts. Não use classes. Exporte um objeto ou funções puras (ex: walletRepository) que recebam os dados e executem prisma.wallet.create. Lembre-se que o balance no Prisma deve nascer igual ao initialBalance informado.

Use Case: Crie ou mova a regra de negócio para src/modules/wallets/use-cases/create-wallet.use-case.ts. Aplique a Injeção de Dependência Funcional: crie uma factory function (ex: makeCreateWalletUseCase(repository)) que retorna a função assíncrona que executa a regra de negócio. Se o initialBalance for menor que zero, lance um erro (regra de negócio simples para teste).

Rota/Controller: Crie ou mova o mapeamento web para src/modules/wallets/http/wallet.routes.ts. Registre a rota POST /wallets. Na definição da rota do Fastify, use a propriedade schema do Swagger associada ao Zod (body, response, tags: ['Wallets']). Instancie o use case passando o repositório real do Prisma e execute-o.

Regras Absolutas:

Paradigma Funcional: É estritamente proibido o uso da palavra reservada class. Tudo deve ser resolvido com funções, closures e objetos literais.

Clean Code e Nomenclatura: Código 100% em Inglês, sem comentários (//), exceto para diretivas de linter. Respeite rigorosamente os nomes dos arquivos com os sufixos .dto.ts, .repository.ts, .use-case.ts e .routes.ts.

Versionamento: Ao finalizar, gere o commit. A mensagem de commit deve ser obrigatoriamente escrita em Inglês.

Dessa forma, o Claude vai pegar aquele código que estava espalhado (src/routes, src/usecases) e unificar tudo cirurgicamente dentro de src/modules/wallets/.
