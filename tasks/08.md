Diretiva de Sistema: Você está operando em Modo de Fluxo de Trabalho Dinâmico (Agentic Mode). Você tem autonomia para planejar e executar o seguinte Épico, limitado a um máximo de 10 sub-tarefas.

Épico: Implementar a Fase 3 (Motor Transacional) para a API do Nummus Premium.
Contexto: Esta é a parte mais crítica do sistema. Vamos criar o módulo de transações (src/modules/transactions/) mantendo a arquitetura de Fatiamento Vertical e Programação Funcional (Zero Classes).

Plano de Execução:

1. DTO (dtos/create-transaction.dto.ts):

Crie o Zod schema createTransactionSchema.

Campos: amount (number), type (enum INCOME, EXPENSE, BALANCE_ADJUSTMENT), paymentMethod (enum CASH, PIX, BANK_TRANSFER, DEBIT_CARD), date (string/date), description (string), walletId (string obrigatória aqui), categoryId (string).

2. Repository (repositories/transaction.repository.ts):

REGRA CRÍTICA DE ATOMICIDADE: Exporte uma função pura que crie a transação e atualize o saldo da carteira (wallet.balance) na mesma operação. Você DEVE usar prisma.$transaction para garantir que o banco não grave a transação se a atualização de saldo falhar.

3. Use Case (use-cases/create-transaction.use-case.ts):

Receba os dados e implemente a matemática:

Se type === 'INCOME', some o amount ao saldo atual da carteira.

Se type === 'EXPENSE', subtraia o amount do saldo da carteira.

Se type === 'BALANCE_ADJUSTMENT', defina o saldo da carteira exatamente igual ao amount fornecido e grave o valor na tabela de transação como a diferença.

Lance erro se a carteira ou categoria não existirem no banco (você precisará injetar métodos de busca desses repositórios ou lidar com o erro do Prisma).

4. Routes & Server (http/transaction.routes.ts e server.ts):

Registre a rota POST /transactions protegida por verifyAuth. Use o Zod schema.

Registre as rotas no server.ts com a tag do Swagger ['Transactions'].

Regras Absolutas:

Zero Classes: Apenas paradigma funcional.

Clean Code: Todo o código (variáveis, arquivos) 100% em Inglês.

Após criar/alterar, rode npx tsc --noEmit internamente.

Ao finalizar e passar no TS, gere um único Git commit em Inglês ("feat(transactions): implement core transactional engine with atomic balance updates"). Reporte de volta.
